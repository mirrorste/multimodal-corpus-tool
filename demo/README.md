## 1. 快速开始

### 1.1 环境要求

- **Python**: 3.10 或更高版本
- **操作系统**: Windows / macOS / Linux
- **网络**: 需要网络连接以测试在线视频下载
- **浏览器**: Chrome / Edge / Firefox（推荐使用 Chrome）

### 1.2 一键启动

#### 方式一：使用初始化脚本（推荐）

```bash
# 进入项目目录
cd /path/to/project

# 运行初始化脚本
python setup_demo.py

# 脚本执行完成后，启动后端服务
cd backend
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

#### 方式二：手动启动

```bash
# 1. 安装依赖
pip install fastapi uvicorn sqlalchemy aiosqlite pydantic pydantic-settings python-multipart aiofiles yt-dlp httpx

# 2. 创建环境配置文件
cd backend
echo "USE_SQLITE=true" > .env
echo "DOWNLOAD_DIR=./demo_downloads" >> .env
echo "UPLOAD_DIR=./demo_uploads" >> .env

# 3. 启动服务
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### 1.3 打开测试页面

服务启动后，在浏览器中打开：

```
demo/demo_test.html
```

或者访问 API 文档：

```
http://127.0.0.1:8000/docs
```

## 4. 常见问题

### 4.1 服务启动问题

**Q: 启动时提示端口被占用？**
A: 修改启动端口：
```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```
然后在测试页面修改 API 地址为 `http://127.0.0.1:8001`

**Q: 提示缺少依赖？**
A: 重新安装依赖：
```bash
pip install fastapi uvicorn sqlalchemy aiosqlite pydantic pydantic-settings python-multipart aiofiles yt-dlp
```

### 4.2 下载问题

**Q: 下载一直处于"排队中"？**
A: 可能是异步任务未正确启动，刷新页面查看状态，或重新触发下载。

**Q: YouTube 视频下载失败？**
A: 
- 检查网络连接是否能访问 YouTube
- 确认视频链接是否有效、是否为公开视频
- 部分视频可能因为版权或地区限制无法下载

**Q: B 站视频下载失败？**
A:
- 确认视频链接格式正确（建议使用 BV 号链接）
- 部分高清视频可能需要登录才能下载

### 4.3 文件上传问题

**Q: 支持哪些视频格式？**
A: 支持 MP4, AVI, MKV, MOV, WEBM, FLV, WMV, M4V 共 8 种格式。

**Q: 上传大文件很慢？**
A: 这是正常现象，文件上传速度受限于磁盘 I/O 和系统性能。2GB 以内的文件都可以正常上传。

### 4.4 数据库问题

**Q: 数据存在哪里？**
A: Demo 版本使用 SQLite 数据库，文件位于 `backend/test_multimodal_corpus.db`。

**Q: 如何重置数据？**
A: 删除数据库文件即可重置：
```bash
rm backend/test_multimodal_corpus.db
```
重启服务后会自动创建新数据库。
